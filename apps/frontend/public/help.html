Quick StartGet up and running in minutes

💡
First time? Start here. This walkthrough covers registration → login → setting up your company workspace.
1

Register Your Company

Visit the /register page. Enter your company name, your email address, and a strong password (must contain uppercase, lowercase, number, and a special character). This creates your company workspace and a Company Admin account for you.

2

Login to Your Workspace

Use the email and password you just registered with on the /login page. You will land on the Dashboard with access to all modules.

3

Create Departments

Go to Departments in the sidebar and create the departments in your organisation (e.g., HR, Engineering, Finance). Departments are required before adding employees.

4

Add Users & Employees

Go to User Management to create user accounts. Then go to Employees to link those users to departments and assign designations and salaries.

5

Start Operations

Now you can mark attendance, manage leave requests, run payroll, manage inventory, and process sales — all from the sidebar navigation.

User ManagementAdmin-only · Manage workspace accounts

⚠️
Only Company Admin and Super Admin roles can access this module. Employees will not see it in the sidebar.
👤

Creating a User

Click Add User at the top right. Fill in First Name, Last Name, Email, Password, and Role. Password must include uppercase, lowercase, number, and special character.

  • Select role: Employee or Company Admin
  • User receives access immediately on creation
  • Email must be unique across the platform
🗑️

Deleting a User

Click the Delete button in the user row. A confirmation prompt will appear. Deleted users are soft-deleted and cannot log in but their data is preserved.

  • Action is irreversible via the UI
  • User data (payroll, attendance) is retained
  • Cannot delete your own account
🏢

DepartmentsOrganise your company structure

What are Departments?

Departments represent teams or divisions in your company. Every employee must belong to a department. Departments are created before employees.

➕

Add Department

Click Add Department, enter a name and optional description. A unique code is automatically generated.

  • Name must be at least 2 characters
  • Description is optional
  • Department code is auto-generated
📊

Department Cards

Each department card shows the name, description, and total employee count. You can edit or delete from the card actions.

  • Employee count auto-updates
  • Cannot delete if employees are assigned
👨‍💼

EmployeesLink users to their operational profiles

✅
An employee profile is separate from a user account. A user must exist before you can create an employee record for them.

Employee Record Fields

Each employee record stores: Employee ID (unique code), Department, Designation (job title), Salary, Joining Date, and Status (Active / Inactive / On Leave).

1

Filter by Department

Use the department dropdown at the top to filter the employee list to a specific department.

2

Status Management

Status can be Active, On Leave, or Inactive. Update status via the edit action.

📅

AttendanceTrack daily check-in & check-out

📝

Mark Attendance

Admins click Mark Attendance. Select an employee, date, status, and optionally the check-in and check-out times.

  • Statuses: Present, Absent, Half Day, Leave
  • Check-in/out times are optional
  • Can add notes for context
📊

View Records

All attendance records are displayed in a table with employee name, date, status badge, and times. Delete individual records if needed.

  • Employees see their own records only
  • Admins see all company records
  • Color-coded status badges

Submit a Request

Select leave type, start date, end date, and provide a reason (minimum 5 characters). Submit the form to create a Pending request.

PayrollProcess monthly salaries and records

How Payroll Works

Payroll records are created monthly per employee. The system stores Base Salary, Bonuses, Deductions, Taxes, and auto-calculates Net Salary = Base + Bonuses − Deductions − Taxes.

➕

Create Payroll Record

Select an employee and month, fill in salary components. The record starts as Draft.

  • Month format: YYYY-MM (e.g., 2024-06)
  • Base salary pulled from employee profile
  • All amounts in default currency
📊

Payroll Status Flow

Status progression: Draft → Approved → Paid. Admins update status via the table action.

  • Employees view their own payslips
  • Filter by status or month

Frequently Asked Questions

I forgot my password — how do I reset it? ▼
Currently, password reset is handled by your Company Admin. Ask them to delete your account and recreate it, or contact system support. A self-service reset flow can be enabled by your administrator.
Why can't I see the Users or Departments menu items? ▼
Those menu items are only visible to Company Admin and Super Admin roles. If you are an Employee, those sections are hidden for security. Contact your admin to change your role if needed.
I get "Employee profile not found" on Leaves or Payroll — what does this mean? ▼
Your user account does not have a linked employee profile yet. Ask your Company Admin to go to the Employees module and create an employee record for your account. Once linked, you will be able to see your own leaves and payslips.
What password requirements must I meet? ▼
Your password must be at least 8 characters and include: one uppercase letter (A–Z), one lowercase letter (a–z), one number (0–9), and one special character from this set: !@#$%^&*
Can I have multiple companies in one account? ▼
Each registration creates one company workspace. For multiple companies, register separate accounts. A Super Admin can manage multiple companies but each company operates as an isolated tenant.
Why am I getting "Request failed with status 429"? ▼
You have hit the API rate limit. In production, the limit is 300 requests per 15 minutes per IP. Wait a few minutes and try again. In development mode the limit is much higher (5,000 requests). If you see this frequently in production, contact your system administrator.
How does the sidebar collapse work? ▼
Click the chevron button (‹) in the top-right of the sidebar header to collapse it to an icon-only view. Click the (›) button to expand it again. On mobile, use the hamburger menu (☰) in the top bar to open and close the sidebar.
Can I edit an approved leave request? ▼
Once a leave is Approved, only the status can be changed by an admin. The dates and leave type cannot be edited. To change the request, cancel it and submit a new one.
